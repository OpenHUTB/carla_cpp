# smoke.test_client
# 功能：可能是针对客户端相关功能进行冒烟测试，检查客户端能否正常连接、发送请求以及接收响应等基础功能是否正常，确保客户端在基本使用场景下不会出现明显问题。

# smoke.test_sync
# 功能：大概率是对同步相关机制进行冒烟测试，比如验证不同组件、模块之间的数据同步是否按照预期进行，是否能准确在规定的时间节点或事件触发时实现同步操作，避免出现数据不一致等情况。

# smoke.test_sensor_determinism
# 功能：用于测试传感器的确定性，也就是检查在相同的初始条件和输入下，传感器每次输出的数据是否具有可重复性和确定性，保证传感器的表现稳定可靠，不会出现随机的、不可预期的结果。

# smoke.test_collision_determinism
# 功能：针对碰撞的确定性进行冒烟测试，检验在特定场景下，碰撞检测、碰撞响应等相关行为是否每次都能按照既定的规则准确发生，确保碰撞相关的逻辑在多次重复测试中具有一致性和可预测性。

# smoke.test_vehicle_physics
# 功能：测试车辆物理特性相关的功能，可能会检查车辆的加速、减速、转向、惯性等物理属性表现是否符合真实物理规律以及游戏或模拟环境中设定的物理模型，验证车辆在各种操作下的运动状态是否合理。

# smoke.test_props_loading
# 功能：对道具加载功能进行冒烟测试，查看游戏或模拟场景中的各种道具（如物品、场景装饰物件等）能否正常从相应资源文件中加载出来，并且加载后的显示、属性等是否正确无误。

# smoke.test_sensor_tick_time
# 功能：主要测试传感器的时间间隔相关特性，即检查传感器每次“滴答”（触发数据采集或更新）的时间是否符合预期要求，保证传感器的数据采集频率稳定，避免因时间间隔异常导致的数据不准确或性能问题。

# smoke.test_map
# 功能：对地图相关功能进行冒烟测试，可能涉及地图的加载、显示是否完整正确，地图中的元素（如道路、地标、区域划分等）是否符合设计要求，以及地图与其他游戏或模拟功能之间的交互是否正常等方面的检查。

# smoke.test_snapshot
# 功能：推测是测试快照功能，比如查看能否准确地在某个时刻对游戏或模拟场景的当前状态进行“拍照”（生成快照），并且快照中记录的各种信息（对象状态、环境数据等）是否完整、准确，同时验证快照后续的使用（如恢复场景等操作）是否正常。

# smoke.test_lidar
# 功能：专门针对激光雷达（Lidar）进行冒烟测试，检测激光雷达的扫描范围、精度、点云数据生成等功能是否正常，确保其能为相关应用（如环境感知、建模等）提供准确可靠的数据。

# smoke.test_streamming
# 功能：应该是对数据流式传输相关功能进行冒烟测试，检查数据在网络或系统内以流的形式传输时，是否存在丢包、卡顿、延迟等问题，保证数据能够流畅、稳定地从源端传输到目标端。

# smoke.test_spawnpoints
# 功能：测试生成点（Spawnpoints）相关功能，可能是验证游戏或模拟环境中物体（如角色、车辆等）的生成位置是否准确，生成规则是否符合设定，以及在不同的生成点进行生成操作时是否能正常触发相应的初始化流程等。

# smoke.test_blueprint
# 功能：针对蓝图（Blueprint）功能进行冒烟测试，检查蓝图所定义的对象模板（如建筑、车辆等的设计蓝图）在实例化、属性配置、功能实现等方面是否正确无误，确保基于蓝图创建的实际对象能符合预期要求。

# smoke.test_collision_sensor
# 功能：聚焦于碰撞传感器的专项冒烟测试，除了检查碰撞传感器能否正常检测到碰撞事件发生外，还可能会验证其检测的灵敏度、检测范围以及反馈给系统的碰撞信息是否准确、详细等方面的情况。

# smoke.test_world
# 功能：对整个游戏或模拟世界相关功能进行冒烟测试，涉及世界的初始化、运行状态维护、各种对象在世界中的交互、世界规则的执行等多个方面，确保整个世界能正常运转，没有明显的逻辑或性能问题。

# smoke.test_determinism
# 功能：从更宽泛的角度测试整个系统的确定性，检查在重复相同的操作、输入相同的初始条件下，整个游戏或模拟系统的输出、状态变化等是否始终保持一致，保证系统具有可靠的可预测性和稳定性。
